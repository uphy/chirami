# Implementation Plan

- [x] 1. セットアップと設定モデルの追加
- [x] 1.1 (P) SwiftHTMLToMarkdown ライブラリを SPM 依存として追加する
  - Package.swift (または project.yml) に SwiftHTMLToMarkdown パッケージを追加
  - ビルドが通ることを確認
  - _Requirements: 3.1_

- [x] 1.2 (P) Smart Paste の設定モデルを追加し、config.yaml で制御可能にする
  - `enabled` (デフォルト: true) と `fetch_url_title` (デフォルト: true) の2つのプロパティを持つ設定モデルを定義
  - snake_case ↔ camelCase のマッピングを既存パターンに準拠して実装
  - グローバル設定に `smart_paste` セクションとして組み込む
  - 設定セクションが存在しない場合のデフォルト値動作を確認
  - YAMLStore のファイル監視により、再起動なしで設定変更が反映されることを確認
  - _Requirements: 6.1, 6.2, 6.3, 6.4_

- [x] 2. コンテンツ種別の判定と変換サービスの実装
- [x] 2.1 クリップボードのコンテンツ種別を判定するロジックを実装する
  - NSPasteboard からコンテンツタイプ一覧と内容を読み取る
  - HTML → URL → JSON → コード → プレーンテキストの優先順位で種別を判定
  - HTML: `.html` ペーストボードタイプの存在で判定
  - URL: テキストが `http://` または `https://` で始まる有効な URL か検証
  - JSON: `JSONSerialization` でパース可能か検証
  - コード: 複数行かつ構文要素 (中括弧、セミコロン、インデントパターン等) の存在で簡易判定
  - いずれにも該当しない場合はプレーンテキストとして扱う
  - _Requirements: 2.1, 2.2_

- [x] 2.2 (P) HTML コンテンツを Markdown に変換する機能を実装する
  - SwiftHTMLToMarkdown の BasicHTML クラスを使用して HTML → Markdown 変換を実行
  - 見出し (`h1`-`h6`) が Markdown 見出しに変換されることを確認
  - ハイパーリンクが `[テキスト](URL)` 形式に変換されることを確認
  - リスト (`ul`/`ol`) が Markdown リスト形式に変換されることを確認
  - 太字・斜体のインライン装飾が `**` / `*` に変換されることを確認
  - パースエラー時はプレーンテキストにフォールバック
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5_

- [x] 2.3 (P) URL を Markdown リンク形式に変換する機能を実装する
  - 単一の URL を `[URL](URL)` 形式の Markdown リンクに変換
  - `fetch_url_title` 設定が無効の場合、URL 自身をリンクテキストとして使用
  - `fetch_url_title` 設定が有効の場合、非同期タイトル取得の必要性を結果に含める
  - _Requirements: 4.1, 4.5_

- [x] 2.4 (P) JSON およびコードスニペットをコードブロックに変換する機能を実装する
  - 有効な JSON 文字列をバッククォート3つの `json` 言語指定付きコードブロックで囲む
  - コードと判定された内容を言語指定なしのコードブロックで囲む
  - _Requirements: 5.1, 5.2_

- [x] 3. URL タイトルの非同期取得機能の実装
- [x] 3.1 URL のページタイトルを非同期で取得し、リンクテキストに設定する機能を実装する
  - URLSession で対象ページの HTML を取得する
  - `<title>` タグまたは `og:title` メタタグからタイトルを抽出
  - タイムアウトを 5 秒に設定
  - ネットワークエラーやパースエラー時は `nil` を返却し、呼び出し元でフォールバック処理を行う
  - _Requirements: 4.2, 4.4_

- [x] 4. エディタへのキーバインド追加と変換テキスト挿入の統合
- [x] 4.1 `Cmd+Shift+V` キーバインドでスマートペーストを発動する
  - 既存のキーボードショートカット処理に `Cmd+Shift+V` の分岐を追加
  - `Cmd+V` は既存の通常ペースト動作をそのまま維持
  - エディタにフォーカスがない場合はスマートペーストを発動しない
  - Smart Paste 設定が無効の場合は通常ペーストにフォールスルー
  - _Requirements: 1.1, 1.2, 1.3, 6.1_

- [x] 4.2 変換結果をエディタに挿入し、Undo/Redo・ファイル保存と連携する
  - 変換サービスを呼び出してクリップボードの内容を Markdown に変換
  - 変換結果を `shouldChangeText` → `replaceCharacters` → `didChangeText` パターンで挿入 (Undo サポート)
  - 挿入後に Markdown 再スタイリングとファイル保存が自動的にトリガーされることを確認
  - _Requirements: 1.2_

- [x] 4.3 URL ペースト時のプレースホルダ挿入と非同期タイトル更新を実装する
  - `fetch_url_title` が有効な場合、まず `[](URL)` をプレースホルダとして即座に挿入
  - 挿入時のテキスト範囲を保持し、タイトル取得完了後にリンクテキスト部分のみを置換
  - タイトル取得に失敗した場合、URL 自身をリンクテキストとして設定
  - _Requirements: 4.2, 4.3, 4.4_
