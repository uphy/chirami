# Implementation Plan

- [x] 1. テンプレート解決ユーティリティの実装
- [x] 1.1 (P) パステンプレートの検出・解決・変換機能の実装
  - パス文字列に `{...}` プレースホルダーが含まれるかを判定する機能を実装する
  - テンプレート文字列を指定日付で解決し、プレースホルダーを日付文字列に置換する機能を実装する
  - 複数のプレースホルダーを含むパス（例: `{yyyy}/{MM}/{dd}.md`）に対応する
  - テンプレートを glob パターンに変換する機能（`{...}` → `*`）を実装する
  - ベースディレクトリからの相対パスがテンプレートにマッチするか判定する機能を実装する（false positive フィルタ用）
  - 各機能のユニットテストを作成する（正常系・異常系・複数プレースホルダー・false positive 排除）
  - _Requirements: 1.1, 1.2, 1.3_
  - _Contracts: PathTemplateResolver Service_

- [x] 1.2 (P) Duration 文字列のパース機能の実装
  - `"2h"` → 7200秒、`"30m"` → 1800秒のような人間可読な duration 文字列を秒数に変換する機能を実装する
  - `nil` や不正値は `0` にフォールバックする
  - ユニットテストを作成する（`"2h"`, `"30m"`, `"0"`, `nil`, 不正値）
  - _Requirements: 1.5_
  - _Contracts: DurationParser Service_

- [x] 2. Config・データモデルの periodic note 対応
- [x] 2.1 (P) ノート設定に rollover_delay と template フィールドを追加
  - ノート設定に `rollover_delay`（遅延時間文字列）と `template`（テンプレートファイルパス）のオプションフィールドを追加する
  - YAML の snake_case キーを CodingKeys でマッピングする
  - パスにテンプレートプレースホルダーが含まれるかを判定する computed property を追加する
  - periodic note ではテンプレート文字列の SHA256 先頭6文字から ID を導出するように noteId を変更する（静的ノートは既存動作を維持）
  - noteId の変更に対するユニットテストを追加する
  - _Requirements: 1.4, 1.5, 1.6_
  - _Contracts: ConfigModels State_

- [x] 2.2 (P) ノートモデルに periodic note 情報を追加
  - テンプレート文字列、rollover delay、テンプレートファイルパスを保持する構造体を定義する
  - ノートモデルにオプショナルプロパティとして追加する（`nil` なら静的ノート）
  - Equatable の実装に新プロパティを反映する
  - _Requirements: 1.1, 1.2_
  - _Contracts: Note State_

- [x] 3. (P) ファイル検索エンジンの実装
  - テンプレートにマッチする既存ファイルの検索・ソート・ナビゲーションユーティリティを実装する
  - テンプレートの静的プレフィックス（`{` より前のパス部分）をベースディレクトリとして使用する
  - 複数階層テンプレートの場合はディレクトリを再帰的に列挙する
  - テンプレートフォーマットで逆パースできるファイルのみをフィルタする（false positive 排除）
  - ファイルの相対パスを lexicographic ソートし順序を決定する
  - 現在のファイルに対する前後のファイルを返す機能を実装する
  - テスト用ディレクトリを使ったユニットテストを作成する（単一階層・複数階層・false positive 混在）
  - _Requirements: 4.2, 4.3, 4.4, 4.5_
  - _Contracts: PeriodicFileNavigator Service_

- [x] 4. NoteStore のテンプレート解決統合
- [x] 4.1 指定日付でのノート解決とファイル作成の実装
  - テンプレートを指定日付で解決し、対応するノートオブジェクトを返す機能を実装する
  - 解決先の親ディレクトリが存在しない場合に自動作成する
  - 解決先のファイルが存在しない場合、テンプレートファイルが指定されていればその内容をコピーして新規作成する
  - テンプレートファイルが未指定の場合は空ファイルを作成する
  - テンプレートファイルが存在しない場合は警告ログを出力し空ファイルにフォールバックする
  - タイトルは「設定タイトル — 解決済みファイル名」の形式で生成する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 3.1_
  - _Contracts: NoteStore Service_

- [x] 4.2 config 読み込み時の periodic note 分岐の統合
  - config 読み込み処理にテンプレート検出ロジックを追加し、periodic note を自動判定する
  - periodic note の場合は論理日時（現在日時 − rollover_delay）でテンプレートを解決する
  - periodic note では security-scoped bookmark による永続的ファイルアクセスをスキップする
  - 静的ノートの処理は既存動作を維持する（後方互換を確認）
  - _Requirements: 1.1, 1.2, 2.1, 2.5_
  - _Contracts: NoteStore Service_

- [x] 5. (P) ナビゲーション UI の実装
  - periodic note のタイトルバーに前 (◀)、次 (▶)、Today (●) のナビゲーションボタンを配置する
  - SF Symbols を使用したコンパクトなボタンを既存のタイトルバー領域に追加する
  - ボタンの有効/無効状態と Today ボタンの表示/非表示を外部から更新できるインターフェースを提供する
  - 静的ノートではナビゲーションボタンを非表示にする
  - _Requirements: 4.1, 4.7, 5.1, 5.3_
  - _Contracts: NotePanel Service_

- [x] 6. ナビゲーションロジックの実装
- [x] 6.1 前後ファイルへのナビゲーション機能の実装
  - ◀ ボタンでソート順の前のファイル、▶ ボタンで次のファイルに切り替える機能を実装する
  - ファイル切り替え時にファイル内容の再ロード、ファイル監視の再設定、タイトルの更新を行う
  - 前後にファイルが存在しない場合は対応するボタンを無効化する
  - ナビゲーション状態（今日のノートを表示中か否か）を管理する
  - periodic note のタイトルは「設定タイトル — 解決済みファイル名」形式で表示し、静的ノートは従来どおりとする
  - _Requirements: 3.1, 3.2, 4.3, 4.4, 4.5, 4.6_
  - _Contracts: NoteWindowController Service, State_

- [x] 6.2 Today ボタンによる今日のノートへの復帰機能の実装
  - 論理日時でテンプレートを再解決し、対応するノートに切り替える機能を実装する
  - 今日のノートを表示中は Today ボタンを非表示にし、別のノートにナビゲートしている場合のみ表示する
  - _Requirements: 5.1, 5.2, 5.3_
  - _Contracts: NoteWindowController Service_

- [x] 7. ロールオーバーの実装
- [x] 7.1 (P) ロールオーバータイマーの管理
  - periodic note が存在する場合に60秒間隔のポーリングタイマーを開始する
  - 各 periodic note コントローラのテンプレートと rollover_delay を参照し、論理日時でパスを再解決する
  - 解決結果が現在表示中のパスと異なった場合にロールオーバーを検出する
  - periodic note がなくなった場合はタイマーを停止する
  - config reload 時に既存コントローラのナビゲーション状態を保持する
  - _Requirements: 6.1, 6.2_
  - _Contracts: WindowManager Service_

- [x] 7.2 (P) ロールオーバー応答処理の実装
  - ロールオーバー検出時、ユーザーが今日のノートを表示中なら新しい解決結果のノートに自動切り替えする
  - ユーザーが別のノートにナビゲートしている場合は現在の表示を維持する（自動切り替えしない）
  - 切り替え時にファイル作成（テンプレートコピーまたは空ファイル）、コンテンツ再ロード、ファイル監視の再設定を行う
  - _Requirements: 6.3, 6.4_
  - _Contracts: NoteWindowController Service_

- [x] 8. ウィンドウ状態保持の確認と統合
  - periodic note のウィンドウ位置・サイズが再起動後も保持されることを確認する（既存の WindowState 永続化の noteId キーがテンプレートベースの安定 ID と整合）
  - ナビゲーション状態（表示中のファイル）が永続化されないことを確認する（再起動時は常に論理日時で解決）
  - 静的ノートと periodic note が混在する config で全機能が正常動作することを統合確認する
  - _Requirements: 7.1, 7.2_
