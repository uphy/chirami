# Implementation Plan

- [x] 1. 添付ファイル保存ディレクトリの設定機能
- [x] 1.1 (P) 設定モデルに添付ディレクトリのフィールドとパス解決ロジックを追加
  - グローバルデフォルト設定とノートごとの設定に `attachments_dir` フィールドを追加する
  - YAML の snake_case キーと Swift の camelCase プロパティのマッピングを既存パターンに従って設定する
  - ノート固有設定 → グローバルデフォルト → フォールバック値の優先順位でディレクトリパスを解決するロジックを実装する
  - 通常ノートのフォールバック: ノートファイル名（拡張子なし）+ `.attachments/` をノートと同じディレクトリに生成
  - Periodic note のフォールバック: テンプレートパスの親ディレクトリ直下に `attachments/` を生成し、全日付で共有する
  - 相対パスはノートの親ディレクトリを基準に解決、`~/` はホームディレクトリに展開、`/` 始まりはそのまま使用
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5, 4.6, 4.7_

- [x] 1.2 Note モデルに添付ディレクトリを追加し、ノート生成時に設定から解決
  - Note 構造体に添付ディレクトリの URL プロパティを追加する
  - 等価比較にも新しいプロパティを含める
  - 静的ノートの生成処理で、設定の解決ロジックを呼び出して添付ディレクトリを設定する
  - Periodic note の生成処理でも同様に、テンプレートパスを考慮した解決ロジックを呼び出す
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5_

- [x] 2. (P) 画像保存サービスの新規作成
  - 画像を PNG 形式でファイルに保存し、Markdown 画像構文テキストを返すサービスを新規作成する
  - 画像データの PNG 変換を行い、SHA256 ハッシュプレフィックス（先頭16バイト = 32文字 hex）でファイル名を生成する
  - 同一コンテンツのハッシュが一致することを利用し、既存ファイルがある場合は書き込みをスキップして重複を防止する
  - 保存先ディレクトリが存在しない場合は自動作成する
  - ノートファイルの位置から添付ディレクトリへの相対パスを計算し、`![](相対パス/image-xxxx.png)` 形式の Markdown テキストを生成する
  - PNG 変換失敗、ディレクトリ作成失敗、ファイル書き込み失敗の各エラーを区別できるエラー型を定義する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 3.2_

- [x] 3. (P) Markdown スタイラーに相対パス解決機能を追加
  - スタイラーにノートファイルの親ディレクトリを保持するプロパティを追加する
  - 画像パターン処理で、相対パス（HTTP URL でも絶対パスでもないパス）をノートの親ディレクトリを基準に絶対パスに解決する
  - 解決済みの絶対パスを画像属性と画像キャッシュの両方に一貫して使用し、レンダリングパイプラインとの整合性を保つ
  - HTTP/HTTPS URL、絶対パス、チルダパスは既存の動作を維持する
  - _Requirements: 5.1, 5.2_

- [x] 4. 画像ペースト処理の統合
- [x] 4.1 MarkdownTextView にクリップボード画像ペースト検出を追加
  - ノートファイルのパスと添付ディレクトリのパスを保持するプロパティを追加する
  - Cmd+V のペースト処理をオーバーライドし、クリップボードに `.tiff` または `.png` タイプの画像データが含まれているかを検出する
  - クリップボードにテキストと画像の両方が存在する場合は、テキストペースト（既存動作）を優先する
  - 画像のみの場合は、画像保存サービスを呼び出して PNG ファイルとして保存し、返された Markdown テキストをカーソル位置に挿入する
  - ノートパスまたは添付ディレクトリが未設定の場合は、画像ペーストを無効とし通常ペーストにフォールバックする
  - エラー発生時もログ出力のみで通常ペーストにフォールバックし、操作フローを中断しない
  - _Requirements: 1.1, 1.2, 1.3, 3.1_

- [x] 4.2 Views 層でノートパスと添付ディレクトリの受け渡しチェーンを構築
  - エディターコンポーネントにノートパスと添付ディレクトリのパラメータを追加する
  - ビュー生成時と更新時にテキストビューのプロパティに値を設定する
  - スタイリング処理でノートの親ディレクトリをスタイラーに設定し、相対パス解決を有効にする
  - ウィンドウのコンテンツビューからエディターにノートのパスと添付ディレクトリを渡す
  - _Requirements: 3.3, 5.1_
