## Context

editor-web は CodeMirror 6 ベースのエディタで、すでに複数の ViewPlugin（tldraw, mermaid, checkbox など）がある。スラッシュコマンドは「行頭 `/` → テキスト入力 → コマンド選択」という UI パターンで、Notion や Linear に近い体験を提供する。

現状、tldraw ブロックや mermaid ブロックは手入力でコードフェンスを書くしかなく、コマンドの発見性がない。

## Goals / Non-Goals

**Goals:**
- 行頭 `/` でコマンドピッカー（ドロップダウン）を表示
- キーボード（↑↓・Enter・Escape）と文字入力によるフィルタリング
- `/tldraw`：ブロック挿入 + 即オーバーレイ open
- `/mermaid`：ブロック挿入 + カーソルをブロック内に配置
- `/table`：Markdownテーブルテンプレート挿入 + カーソルをヘッダセルに配置
- 将来のコマンド追加が容易な拡張可能設計

**Non-Goals:**
- マウスホバーによる詳細プレビュー
- コマンドのカテゴリ分類・グループ化（初期は不要）
- コマンドのカスタマイズ・ユーザー定義
- ネスト（サブコマンド）

## Decisions

### トリガー条件

**決定**: `/` が行の先頭（前に空白のみ許可しない = 行頭のみ）に入力された場合のみ発動する。

- 理由: URL（`https://...`）や通常テキスト内の `/` と衝突しないようにするため、行頭のみに限定する。
- 実装: `ViewPlugin.update()` で `docChanged` かつ入力が `/` のときに行先頭チェックを行う。

### ピッカーの実装方法

**決定**: DOM で直接ポップアップを構築する（React 不使用）。

- 理由: コマンドリストは静的かつシンプルで React が不要。tldraw のオーバーレイとは異なり、軽量な DOM 操作で十分。
- ポップアップは `document.body` に append し、カーソル位置の `getBoundingClientRect()` を基に `position: fixed` で配置する。

### コマンドの挿入後挙動

**決定**: コマンドごとに異なる `execute(view)` 関数を持つ設計にする。

- `/tldraw`: `\`\`\`tldraw\n\n\`\`\`` を挿入後、`openTldrawOverlay` を呼び出す
- `/mermaid`: `\`\`\`mermaid\n\n\`\`\`` を挿入し、カーソルを内側の空行に置く
- `/table`: `| | |\n|---|---|\n| | |` を挿入し、カーソルを1行目の最初のセルに置く

### スラッシュとフィルタテキストの扱い

**決定**: パススルー方式を採用する。入力文字は常にエディタに挿入され、ピッカーはエディタの現在行を読んでフィルタする。

- `/` を入力 → エディタに `/` が挿入され、ピッカーが開く。
- 続けて文字を入力 → エディタに `/xxx` が挿入される。ピッカーは行頭から現在カーソルまでの文字列を読んでフィルタリングする。
- マッチなし → ピッカーが自動的に閉じる。`/xxx` はエディタにそのまま残る。
- Escape またはフォーカス移動 → ピッカーが閉じる。`/xxx` はエディタにそのまま残る。
- Enter でコマンド選択 → 行頭の `/xxx`（`/` からカーソル直前まで）をブロックテキストで置換する。
- **代替案（却下）**: キーストロークをキャプチャしてフィルタとして使い、キャンセル時は `/` のみ残す方式。入力した文字が失われるためユーザーが意図しない入力消失を体験しやすく却下。

## Risks / Trade-offs

- **ポップアップ位置のずれ** → WKWebView 内での `getBoundingClientRect()` は通常通り動作するが、スクロール位置や CSS transform に注意。カーソル行の `top` + `height` を基準に配置し、画面下端はみ出し時は上方向に表示する。
- **`/` を単体で入力したいケース** → ピッカーが開いてしまうが、Escape で即キャンセルできる。通常テキスト内では発動しない（行頭のみ）ため影響は限定的。
- **他の Markdown 記法との干渉** → 行頭 `/` は標準 Markdown に存在しないため衝突なし。
