# Slash Command

## Purpose

エディタでスラッシュ（`/`）を行頭に入力することでコマンドピッカーを呼び出し、Markdown ブロックや要素を素早く挿入できる機能。

## Requirements

### Requirement: 行頭スラッシュでコマンドピッカーを表示

エディタは行頭（インデントなし）で `/` を入力した場合にコマンドピッカーを表示しなければならない（SHALL）。ピッカーはカーソル位置の直下に表示され、利用可能なコマンドの一覧を表示する。

#### Scenario: 行頭でスラッシュを入力した場合

- **WHEN** ユーザーが空行またはその行の先頭で `/` を入力する
- **THEN** コマンドピッカーがカーソルの近くに表示され、利用可能なコマンド一覧が表示される

#### Scenario: 行の途中でスラッシュを入力した場合

- **WHEN** ユーザーが行の途中（先頭以外）で `/` を入力する
- **THEN** コマンドピッカーは表示されない

### Requirement: コマンドのキーボードフィルタリング

ピッカーが表示されている間、ユーザーが文字を入力するとコマンドリストがリアルタイムにフィルタリングされなければならない（SHALL）。

#### Scenario: 文字入力によるフィルタリング

- **WHEN** ピッカーが開いている状態でユーザーが文字を入力する
- **THEN** 入力文字にマッチするコマンドのみが一覧に表示される

#### Scenario: フィルタ結果がゼロの場合

- **WHEN** ピッカーが開いている状態でどのコマンドにもマッチしない文字列になる
- **THEN** コマンドピッカーが自動的に閉じ、エディタには `/` を含む入力済みの文字列がそのまま残る

### Requirement: コマンドのキーボードナビゲーションと選択

ピッカーが表示されている間、キーボードでコマンドを選択できなければならない（SHALL）。

#### Scenario: 矢印キーでの選択移動

- **WHEN** ピッカーが開いている状態で ↑ / ↓ キーを押す
- **THEN** ハイライトされるコマンドが上下に移動する

#### Scenario: Enter キーでのコマンド実行

- **WHEN** ピッカーでコマンドがハイライトされた状態で Enter キーを押す
- **THEN** ピッカーが閉じ、コマンドに対応するブロックがエディタに挿入される

#### Scenario: Escape キーでのキャンセル

- **WHEN** ピッカーが開いている状態で Escape キーを押す
- **THEN** ピッカーが閉じ、エディタには `/` およびそれまでに入力した文字がそのまま残る

### Requirement: tldraw ブロックの挿入とオーバーレイ起動

`/tldraw` コマンドを選択すると、tldraw コードブロックを挿入してすぐにオーバーレイエディタを開かなければならない（SHALL）。

#### Scenario: tldraw コマンドの実行

- **WHEN** ユーザーがコマンドピッカーで `/tldraw` を選択する
- **THEN** `\`\`\`tldraw\n\`\`\`` ブロックが挿入され、直ちに tldraw オーバーレイエディタが開く

### Requirement: mermaid ブロックの挿入

`/mermaid` コマンドを選択すると、mermaid コードブロックを挿入してカーソルをブロック内に配置しなければならない（SHALL）。

#### Scenario: mermaid コマンドの実行

- **WHEN** ユーザーがコマンドピッカーで `/mermaid` を選択する
- **THEN** `\`\`\`mermaid\n\n\`\`\`` ブロックが挿入され、カーソルがブロック内の空行に置かれる

### Requirement: テーブルテンプレートの挿入

`/table` コマンドを選択すると、Markdown テーブルテンプレートを挿入してカーソルを先頭セルに配置しなければならない（SHALL）。

#### Scenario: table コマンドの実行

- **WHEN** ユーザーがコマンドピッカーで `/table` を選択する
- **THEN** 2列2行のテーブルテンプレートが挿入され、カーソルが1行目の最初のセル内に置かれる
